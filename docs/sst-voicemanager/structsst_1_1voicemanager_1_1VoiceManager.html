◆ MonoPriorityMode

template<typename Cfg , typename Responder , typename MonoResponder >
enum struct MonoPriorityMode
strong

MonoPriorityMode determines which note "wins" in MONO_NOTES and MONO_LEGATO modes when a new note-on arrives while a voice is already playing.

LATEST: the new note always takes over (default). HIGHEST: the new note only takes over if its key is strictly higher than the current voice. LOWEST: the new note only takes over if its key is strictly lower than the current voice.

template<typename Cfg , typename Responder , typename MonoResponder >
void guaranteePort ( int16_t  port)

Pre-allocate the per-port voice-tracking state for port.

Port 0 is always pre-allocated. For any other port this MUST be called (off the audio thread) before sending note events to that port: the note-event API is tagged non-blocking and will not lazily create the per-port state, so a note on an unguaranteed port would violate that constraint and allocate.

template<typename Cfg , typename Responder , typename MonoResponder >
bool setPolyphonyGroupParent ( uint64_t  childGroup,
uint64_t  parentGroup 
)

Make childGroup count its voices against parentGroup (and every ancestor above it), forming a hierarchy.

A voice still belongs to exactly one group but is budgeted by the whole ancestor chain. Pass noPolyphonyGroupParent to detach. Returns false (no-op) if it would create a cycle, or if the parent is MONO (only leaf groups may be MONO).
