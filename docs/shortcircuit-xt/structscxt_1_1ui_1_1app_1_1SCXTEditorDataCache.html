Detailed Description

The iriginal design of the SC UI (which is still partly that way) has local copies of the data cache right next to the particular panel.

This works well until you want to share data across more than one panel, and then it gets clumsy, or if your heirarchy gets deep.

So this class exists to have a place for the SCXTEditorResonder thingy to push data copies and for hte UI to bind to, since the UI will reliably be components which haveeditor.
